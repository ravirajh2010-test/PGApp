# PG Stay - Render Deployment Guide

Complete guide to deploy your application to Render.com

## Prerequisites

1. **GitHub Account** - Your project must be pushed to GitHub
2. **Render Account** - Free account at https://render.com
3. **Email Configuration** - Gmail app password for sending emails

---

## Step 1: Prepare Your GitHub Repository

### 1.1 Create GitHub Repository
```bash
# If not already done
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/yourusername/pg-stay.git
git push -u origin main
```

### 1.2 Ensure .gitignore is Configured
Create `.gitignore` in the root directory:
```
node_modules/
.env
.env.local
.env.production.local
dist/
build/
.DS_Store
*.log
*.code-search
.vscode/
```

## Step 2: Set Up Render Account

### 2.1 Sign Up for Render
1. Go to https://render.com
2. Sign up with GitHub
3. Authorize Render to access your repositories

---

## Step 3: Deploy Using render.yaml (Recommended)

### 3.1 Deploy with Infrastructure File
This is the easiest method - Render will automatically detect `render.yaml`

1. Go to https://dashboard.render.com
2. Click **"New +"** → **"Blueprint"**
3. Select your repository
4. Render will auto-detect `render.yaml`
5. Click **"Create Blueprint"**

### 3.2 Configure Environment Variables

Render will create three services:
- **pg-stay-backend** (Node.js server)
- **pg-stay-frontend** (React app)
- **pg-stay-db** (PostgreSQL database)

For each service, configure these environment variables in the Render dashboard:

#### Backend Service: `pg-stay-backend`
Environment Variables to add:
```
EMAIL_USER=your_gmail@gmail.com
EMAIL_PASSWORD=your_app_password
RAZORPAY_KEY_ID=your_key
RAZORPAY_KEY_SECRET=your_secret
```

(These are auto-added)
```
NODE_ENV=production
DATABASE_URL=<auto-set by Render PostgreSQL>
JWT_SECRET=<auto-generated by Render>
```

#### Frontend Service: `pg-stay-frontend`
```
VITE_API_URL=https://pg-stay-backend.onrender.com/api
```
(Get the actual URL from your deployed backend service)

## Step 4: Configure Email Service (Important!)

### 4.1 Gmail App Password Setup
Gmail doesn't allow direct password access. Set up an App Password:

1. Go to https://myaccount.google.com/apppasswords
2. Select "Mail" → "Windows Computer" (or your OS)
3. Generate app password (16 characters)
4. Copy this password to `EMAIL_PASSWORD` in Render

**Example:**
```
EMAIL_USER=ravirajh2010@gmail.com
EMAIL_PASSWORD=abcd efgh ijkl mnop (with spaces)
```

## Step 5: Database Initialization

### 5.1 Render PostgreSQL Setup
After deployment starts, the database will be created automatically.

### 5.2 Initialize Database Schema
The database schema will be created from your `database/schema.sql` when needed.

**Option 1: Automatic (Recommended)**
- Render will run migrations via backend startup
- No manual action needed

**Option 2: Manual via Render Console**
1. Go to Render Dashboard → pg-stay-db
2. Click "Database" → "Browser"
3. Upload/execute SQL from `database/schema.sql`

## Step 6: Manual Deployment (If render.yaml doesn't work)

If automatic deployment fails, deploy manually:

### 6.1 Deploy Backend
1. Go to https://dashboard.render.com
2. Click **"New +"** → **"Web Service"**
3. Select your GitHub repo
4. Configure:
   - **Name:** pg-stay-backend
   - **Branch:** main
   - **Build Command:** `cd backend && npm install`
   - **Start Command:** `cd backend && npm start`
   - **Environment:** Node
   - **Plan:** Free
5. Add environment variables (see Step 4)
6. Create Web Service

### 6.2 Add PostgreSQL Database
1. In same Render project, click **"New +"** → **"PostgreSQL"**
2. Configure:
   - **Name:** pg-stay-db
   - **Database Name:** pg_stay_production
   - **User:** pg_stay_user
   - **Region:** Choose closest to you
   - **Plan:** Free
3. Create database
4. Note the connection string - add to backend service

### 6.3 Deploy Frontend
1. Click **"New +"** → **"Static Site"** (or Web Service)
2. Select your GitHub repo
3. Configure:
   - **Name:** pg-stay-frontend
   - **Branch:** main
   - **Build Command:** `cd frontend && npm install && npm run build`
   - **Publish Directory:** `frontend/dist`
4. Add environment variable:
   - `VITE_API_URL=https://pg-stay-backend.onrender.com/api`
5. Deploy

## Step 7: Update API Configuration

Update `frontend/src/services/api.js`:

```javascript
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:5000/api';

const api = {
  get: (url) => fetch(`${API_BASE_URL}${url}`).then(r => r.json()),
  post: (url, data) => fetch(`${API_BASE_URL}${url}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(data)
  }).then(r => r.json()),
  // ... other methods
};
```

## Troubleshooting

### Issue: "Cannot connect to database"
**Solution:**
1. Check DATABASE_URL in backend environment variables
2. Verify PostgreSQL service is running (green status)
3. Check backend logs for connection errors

### Issue: "Frontend can't reach backend"
**Solution:**
1. Check VITE_API_URL is correct
2. Check CORS is enabled in backend (it should be)
3. Clear browser cache and reload

### Issue: "Email not sending"
**Solution:**
1. Verify EMAIL_USER and EMAIL_PASSWORD are correct
2. Gmail might need you to approve new login location
3. Check backend logs for email errors

### Issue: "Free tier running out of resources"
**Solution:**
1. Upgrade to Starter plan ($7/month)
2. Or optimize database queries
3. Render allocates resources fairly

## FAQ

**Q: Can I use my own domain?**
A: Yes! Go to Service Settings → Custom Domain → Add your domain

**Q: How do I backup my database?**
A: Render auto-backups. Go to Database → Backups

**Q: Can I scale later?**
A: Yes! Upgrade plan anytime from Free → Starter → Standard

**Q: Is my data secure?**
A: Yes! Render provides:
- Automatic SSL certificates
- Encrypted connections
- Daily backups
- DDoS protection
